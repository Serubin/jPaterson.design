Point Cloud to Mesh - Virtual Mask

CS 184: Computer Graphics and Imaging, Spring 2017

Jesse Paterson & Bryce Schmidtchen

Summary:

This project aims to make virtual masks that will mirror the faces of users in real life. We will begin with a point cloud of a face generated from a Kinect. We will convert this point cloud to a mesh using Poisson Surface Reconstruction. The mesh will then be synced with the location of a VR user's face and mirrored back to them so that they can see the results. This project will also enable us to be able to collect the point cloud data of any object or location and virtualize it.

Problem Description

Creating an appropriate virtual avatar for a user has long been a challenge for Virtual Reality. For the purposes of instantly recognizing others in a virtual setting, it is useful to have an avatar that looks similar to users in real life. Moreover, the ability to take real scenes' point cloud data and reconstruct them easily in virtual reality is a problem which could provide interesting abilities to many domains when solved in an accessible way, such as enabling architects to virtually stand in a room that's on the other side of the world or scanning famous locations for people to explore. This project is largely challenging due to the sequence of integrations that must come together to create the entire pipeline from point cloud data collection to virtual reality visualization. Technically, the most challenging part will be getting the our Poisson surface reconstruction algorithm to function and much of the other sections of the project will be integrating systems with the input and output of this algortihm.



Goals and Deliverables

At the very least, we plan to have a beautiful point cloud to mesh solution that will allow us to render point clouds using the Poisson surface reconstruction. From this implementation, we will be able to render surface meshes from point cloud datasets sourced from online. This will be done through an OpenGL rasterizer. The demo would include the demonstration of the rendering of the original point cloud and then the surface mesh after being run through the reconstruction algorithm. We would run this on many different shapes and display these renders in order to demonstrate the integrity of our implementation. The metrics that would determine the success of this minimum viable project would be how well our implementation processes the point cloud data and outputs a mesh.


In our ideal case, we hope to have developed an entire point cloud to VR visualization solution. This will allow us to scan and collect the point cloud of a humanâ€™s face or any other object in reality, input this data into our implementation of the Poisson surface reconstruction, and then visualize the constructed mesh into a virtual reality environment through the HTC Vive. The resulting demonstration that would result from this outcome of the project would be a real-time VR application which rendered the meshes that were reconstructed from the point clouds using our construction algorithm and collected point cloud data. The quality of this system will be again defined largely by how well our surface reconstruction implementation processes the point cloud data and outputs a mesh. Additionally, the success of the project will also depend and be defined by how well the point cloud to mesh reconstruction algorithm is implemented with both ends of the pipeline, including the kinect point cloud data collection as well as the OpenGL renderer / VR integration.

Schedule

We will start on the center of the project, working to acheive a functional point-cloud to mesh algorithm using the Poisson Surface Reconstruction. This phase of the project will involve additional research and comparison to existing solutions.

2nd Week (April 16th - April 22)

3rd Week

We will start collecting Kinect data on faces using ICP and testing our algorithm on a real world model. During this time period, we will continue to polish our reconstruction algorithm as we begin to see messier results. Using real world data will provide its own challenges, but will have more rewarding results.

4th Week

We will start integrating our project into VR using OpenVR. We will use an OpenGL renderer to create a mirror and mask experience.

Resources

This project will be completed on Windows PCs and our virtual reality visualization will be done using the HTC Vive VR HMD. Furthermore, we plan to utilize the Microsoft Kinect sensor to collect our point cloud data that will be then reconstructed into a mesh. The Poisson algorithm that we plan to implement for the point cloud reconstruction is based on this paper on Poisson Surface Reconstruction.


In order to ensure that our point cloud reconstructs to a proper mesh, we may have to implement Iterative Closest Point ourselves so that we can minimize the difference between two point clouds that we collect. If we are unable to implement this ourselves, we may use a C++ library such as this to assist us with the merging of point clouds. Furthermore, we will need to integrate our OpenGL renderer with the HTC Vive. This will be done with the OpenVR API which enable this to be done. There are some resources online that detail what need to be done in order to achieve this integration such as this one. We plan to accomplish this ourselves but may look to these resources for guidance.
